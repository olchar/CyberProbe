📡 Protocol Details

Hop From → To Transport What's Sent Auth
① You → Copilot VS Code IPC Natural language prompt GitHub session
② Copilot → MCP Client JSON-RPC 2.0 {"method": "tools/call", "params": {"name": "ListIncidents", "arguments": {...}}} Internal
③ MCP Client → MCP Server HTTPS (SSE) or stdio Same JSON-RPC, routed by mcp.json URL Azure AD token (automatic)
④ MCP Server → Backend API REST / HTTPS Native API call: GET /security/incidents?$top=10 Service principal / delegated
⑤ API → MCP Server REST response Raw JSON payload —
⑥ MCP Server → MCP Client HTTPS (SSE) or stdio {"content": [{"type": "text", "text": "..."}]} —
⑦ MCP Client → Copilot JSON-RPC 2.0 Tool result injected into LLM context Internal
⑧ Copilot → You VS Code IPC Formatted response in chat panel —

⚙️ Configuration — .vscode/mcp.json

Key insight: The MCP server acts as an adapter between the MCP protocol (JSON-RPC) and the native REST API. Microsoft hosts MCP servers for their APIs — you just point to the URL.
// .vscode/mcp.json
{
  "servers": {
    "Triage": {
      "url": "https://sentinel.microsoft.com/mcp/triage",  // ← Remote MCP server
      "type": "http"                                       // ← HTTPS + SSE transport
    }
  }
}

🔄 Two Transport Modes

🌐 Remote (HTTP + SSE)
Used by: Triage, Data Exploration, Sentinel Graph, Azure, Microsoft Learn, GitHub

How: VS Code sends JSON-RPC over HTTPS to Microsoft's hosted MCP server endpoint. Responses stream back via Server-Sent Events (SSE).

Auth: Azure AD token attached automatically by VS Code. The MCP server's service principal calls the backend API on your behalf.
🖥️ Local (stdio)
Used by: (None currently configured)

How: VS Code spawns a local child process. JSON-RPC messages flow over stdin/stdout pipes. No network involved.

Auth: No Azure AD needed — runs with your local permissions. Can call external APIs using keys from config.json.
